# world

A hand-drawn 3D portfolio world, live at **[subhayu.in/world](https://subhayu.in/world/)**.

Built with React Three Fiber on a "pencil & pastel" procedural art style: every
texture is drawn on a canvas at runtime (no image assets for the art), all text
is hand-lettered via locally hosted fonts, and the whole site works under a
same-origin CSP — no CDNs, no external requests.

## What's inside

- **Entrance** — a brick facade with a double door, a window with a shy
  occupant, and a cat that watches your cursor.
- **Corridor** — an endless hallway of doors, one per room.
- **Journey** — fly a paper plane through the career timeline. Scroll back to
  rewind. It loops forever.
- **Registry** — open-source packages as parcels in an endless column, with
  live-ish PyPI download counts.
- **Warehouse** — client work on easel boards down an endless hall.
- **Contact** — a pier with signposts and a lighthouse.

## Relationship to [subhayu99.github.io](https://github.com/subhayu99/subhayu99.github.io)

This repo is intentionally standalone so the main portfolio (and the template
forkers use) is unaffected by it. It shares that repo's data shape:

- `world.yaml` — the world's own content (rooms, beats, exhibits, easter eggs).
- `public/data/resume.json` — a snapshot generated by the portfolio repo's
  resume pipeline. To refresh: rebuild the portfolio, then
  `cp ../subhayu99.github.io/client/public/data/resume.json public/data/`.
- `public/data/pypi-stats.json` — snapshot of the portfolio repo's PyPI stats
  cache; refresh the same way.

`scripts/generate-world.js` compiles `world.yaml` + `resume.json` into
`public/data/world.json` at build time (it runs automatically before
`dev`/`build`).

## Develop

```bash
npm install
npm run dev          # localhost:5173/world/
npm test             # vitest (happy-dom)
npm run type-check
npm run build && npm run preview
```

## Deploy

Pushes to `main` build and deploy via GitHub Actions to GitHub Pages
(`.github/workflows/deploy.yml`). Because the user site carries the
`subhayu.in` custom domain, this project site is served at
`subhayu.in/world/` automatically (Vite `base` is `/world/`).

## Sounds

See `public/sounds/README.md` for provenance. The audio engine synthesizes
every sound procedurally as a fallback, so removing any file never breaks
playback.
